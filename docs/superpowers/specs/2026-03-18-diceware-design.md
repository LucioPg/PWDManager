# Diceware Password Generation - Design Spec

## Overview

Add a Diceware passphrase generation method alongside the existing random password generator. The "Suggest" button becomes a DaisyUI dropdown with two options: "Password Casuale" (current method) and "Diceware". Diceware settings are stored per-user in a new database table and configurable via the Settings page.

## Dependencies

- `diceware = { git = "https://github.com/LucioPg/diceware.git" }` — Diceware passphrase generation library
- `sys-locale` — System locale detection for default language

## Database Schema

New table `diceware_generation_settings` — created via raw SQL in `init_queries.rs` (same pattern as existing tables), with CRUD operations generated by `sqlx-template` derive macros:

```sql
CREATE TABLE IF NOT EXISTS diceware_generation_settings (
    id INTEGER PRIMARY KEY,
    settings_id INTEGER NOT NULL,
    word_count INTEGER NOT NULL DEFAULT 6 CHECK (word_count >= 1 AND word_count <= 20),
    special_chars INTEGER NOT NULL DEFAULT 0 CHECK (special_chars >= 0),
    force_special_chars BOOLEAN NOT NULL DEFAULT 0,
    numbers INTEGER NOT NULL DEFAULT 0 CHECK (numbers >= 0),
    language TEXT NOT NULL DEFAULT 'EN',
    FOREIGN KEY(settings_id) REFERENCES user_settings(id) ON DELETE CASCADE,
    CHECK (language IN ('EN', 'IT', 'FR')),
    UNIQUE (settings_id)
);
```

**Note:** `UNIQUE (settings_id)` is a separate table constraint at the end, matching the existing `passwords_generation_settings` pattern.

### Local Enum for DB Language Field

Since `diceware::EmbeddedList` (external crate) cannot have sqlx derives added, a local wrapper enum is used:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
#[derive(sqlx::Type)]
#[sqlx(type_name = "TEXT", rename_all = "UPPERCASE")]
pub enum DicewareLanguage {
    EN,
    FR,
    IT,
}

impl From<DicewareLanguage> for diceware::EmbeddedList {
    fn from(lang: DicewareLanguage) -> Self {
        match lang {
            DicewareLanguage::EN => diceware::EmbeddedList::EN,
            DicewareLanguage::FR => diceware::EmbeddedList::FR,
            DicewareLanguage::IT => diceware::EmbeddedList::IT,
        }
    }
}

impl From<diceware::EmbeddedList> for DicewareLanguage {
    fn from(list: diceware::EmbeddedList) -> Self {
        match list {
            diceware::EmbeddedList::EN => DicewareLanguage::EN,
            diceware::EmbeddedList::FR => DicewareLanguage::FR,
            diceware::EmbeddedList::IT => DicewareLanguage::IT,
        }
    }
}
```

### Rust Struct with sqlx-template

Placed in `src/backend/settings_types.rs` (alongside `UserSettings`), not in `db_backend.rs`:

```rust
#[derive(sqlx::FromRow, Debug, Clone, Default, sqlx_template::SqlxTemplate, PartialEq)]
#[table("diceware_generation_settings")]
#[db("sqlite")]
#[tp_upsert(by = "id")]
#[tp_select_builder]
pub struct DicewareGenerationSettings {
    pub id: Option<i64>,
    pub settings_id: i64,
    pub word_count: i32,
    pub special_chars: i32,
    pub force_special_chars: bool,
    pub numbers: i32,
    pub language: DicewareLanguage,
}
```

- `settings_id UNIQUE` — 1:1 relationship with `user_settings`, same pattern as `passwords_generation_settings`
- `language` uses the local `DicewareLanguage` enum with sqlx `Type` derive — no String conversion needed
- CRUD methods are static (e.g., `DicewareGenerationSettings::upsert_by_id(&settings, pool)`, `DicewareGenerationSettings::select_by_settings_id(settings_id, pool)`)
- A default row is created at user registration time, following the same pattern as the existing `passwords_generation_settings` table insertion
- `language` default is overridden by system locale detection at registration

## Backend Generation Logic

### Types

New struct `DicewareGenConfig` (in `backend/password_utils.rs`):

```rust
pub struct DicewareGenConfig {
    pub word_count: usize,
    pub special_chars: u8,
    pub force_special_chars: bool,
    pub numbers: u8,
    pub language: diceware::EmbeddedList,
}
```

Conversion from `DicewareGenerationSettings` → `DicewareGenConfig` via `Into` impl (using the `DicewareLanguage → EmbeddedList` conversion).

### Generation Function

New function in `backend/password_utils.rs`:

```rust
pub fn generate_diceware_password(config: DicewareGenConfig) -> SecretString
```

**Retry loop logic (max 200 iterations, panics if exceeded as safety guard):**

1. Build `diceware::Config`:
   - `.with_embedded(config.language)`
   - `.with_words(config.word_count)`
   - `.with_camel_case(true)` — always enabled
   - If `config.force_special_chars && config.special_chars >= 1` → `.with_special_chars(true)`
2. Call `diceware::make_passphrase()` to generate a passphrase
3. Validate against criteria:
   - `special_chars == 0` → no special characters allowed → retry if any present
   - `special_chars >= 1` → at least `special_chars` special characters required → retry if insufficient
   - `numbers == 0` → no purely numeric words allowed → retry if any present
   - `numbers >= 1` → at least `numbers` numeric words required → retry if insufficient
4. Return `SecretString` if all criteria pass, otherwise retry from step 1
5. If 200 iterations are reached without a valid passphrase, panic with a descriptive message (indicates impossible configuration, e.g., `special_chars = 100, word_count = 2`)

Validation counts special characters and numeric words by splitting the CamelCase passphrase at uppercase boundaries.

**Note on numeric words:** The Diceware word lists include numeric entries (e.g., the Italian list starts with "0", "1", "2", etc.). The `numbers` field controls whether these entries are allowed or required in the generated passphrase — it does NOT inject digits into alphabetic words.

## App Wrapper Integration

### Unified Callback

In `PasswordHandler` (`src/components/globals/password_handler/component.rs`):

Single `on_suggest_method` callback that dispatches based on `GenerationMethod`:

- `GenerationMethod::Random` → calls `generate_suggested_password()` (existing)
- `GenerationMethod::Diceware` → fetches `DicewareGenerationSettings` from DB, converts to `DicewareGenConfig`, calls `generate_diceware_password()`

### DB Functions

CRUD operations generated by `sqlx-template` on `DicewareGenerationSettings` (in `settings_types.rs`):
- `DicewareGenerationSettings::upsert_by_id(&settings, pool)` — insert or update
- `DicewareGenerationSettings::select_by_settings_id(settings_id, pool)` — fetch by settings_id (generated by `#[tp_select_builder]`)

A `From<DicewareGenerationSettings> for DicewareGenConfig` impl handles the conversion.

### User Registration

At user registration, insert a default row into `diceware_generation_settings` alongside the existing `passwords_generation_settings` insertion, following the same pattern. Language defaults to system-detected locale via `detect_system_language()`.

## Settings Page

New section inside the existing "Security" tab (alongside current password generation settings).

### Editable Fields

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| Word count | number input (min 1, max 20) | 6 | |
| Special chars | number input (min 0) | 0 | |
| Force special chars | toggle | OFF | Disabled when `special_chars == 0` |
| Numbers | number input (min 0) | 0 | |
| Language | select (IT/FR/EN) | system-detected | |

### Not Shown in UI

- **CamelCase**: always ON, not toggleable

### Frontend Validation

- `word_count` >= 1 and <= 20
- `special_chars >= 0`
- `numbers >= 0`
- If `special_chars == 0` → disable "Force special chars" toggle
- Backend also defensively handles `force_special_chars == true && special_chars == 0` (treats as if force is OFF)

### Styling

- Dark theme classes coherent with existing settings page
- Follow `pwd-` prefix convention for custom CSS classes
- Classes registered in `assets/input_main.css`

### Save Logic

Form submit → upsert the row in `diceware_generation_settings`, same pattern as existing `passwords_generation_settings` save.

## System Language Detection

New utility function (in `backend/password_utils.rs` or a dedicated module):

```rust
fn detect_system_language() -> DicewareLanguage {
    let locale = sys_locale::get_locale().unwrap_or_default().to_lowercase();
    if locale.starts_with("it") {
        DicewareLanguage::IT
    } else if locale.starts_with("fr") {
        DicewareLanguage::FR
    } else {
        DicewareLanguage::EN
    }
}
```

Called only at user registration to set the default language. `sys_locale::get_locale()` is non-blocking on all supported platforms. After registration, the user can change it in Settings.
